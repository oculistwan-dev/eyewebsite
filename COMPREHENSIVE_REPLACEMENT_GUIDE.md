# 🌍 Eyesite 全站日文替换指南

## 📋 项目概述

本指南提供两种方案来将 eyesite 网站的所有日文内容替换为**适合中国大陆网民的简体中文**，包括：
- 医学术语的标准化（近視→近视、視能訓練士→视光师等）
- 常见日语短语的自然翻译（ホーム→首页、初めての方へ→首次就诊指南等）
- 网络用语和文言文的现代化表达

## 🔄 修改回滚

如果需要回滚修改，有三种方式：

### 方式 1：Git 回滚（推荐）
```bash

git reset --hard HEAD~1

# 或者回滚到指定提交
git reset --hard <commit_hash>
```

### 方式 2：使用备份文件
```bash
# 恢复单个文件
cp topics/index.html.bak topics/index.html

# 或恢复全部文件
find . -name "*.bak" -type f -exec bash -c 'mv "$0" "${0%.bak}"' {} \;
```

### 方式 3：使用 git 命令恢复特定文件
```bash
git checkout HEAD~1 -- topics/index.html
```

## ⚠️ 注意事项

### 重要提醒
1. **备份保留**: 所有原始 `.bak` 文件将被保留，可随时恢复
2. **Git 历史**: 修改会记录在 Git 历史中，可随时回滚
3. **正则匹配**: 脚本使用精确字符串匹配，不会误伤无关内容
4. **编码保证**: UTF-8 编码完全兼容，不会出现乱码

### 可能的问题
| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 某些日文未被替换 | 混淆的混合字符 | 检查原始文本，添加规则 |
| 替换后文本错位 | 字符编码问题 | 确保文件编码为 UTF-8 |
| 性能过慢 | 文件过多 | 正常，脚本会完成（47 个文件通常 < 2 秒） |

## 💡 扩展使用

### 自定义替换规则
编辑 Python 脚本中的 `REPLACEMENT_MAP` 字典：

```python
REPLACEMENT_MAP = {
    "新的日文": "新的中文",
    "另一个日文": "另一个中文",
    # ...更多规则
}
```

编辑 Bash 脚本，添加新的 `sed` 命令：

```bash
sed -i '' 's/新的日文/新的中文/g' "$file"
```

### 与其他脚本集成
两个脚本都支持管道和脚本调用：

```bash
# 作为其他脚本的一部分
python3 comprehensive-japanese-replacement.py && git add . && git commit -m "..."
```
